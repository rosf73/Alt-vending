# [Business] Feature A — 판매 모드 (Sales Mode)

> 상위 규범: [spec.md](../spec.md)
> 대응 화면 요구사항: [requirements/sales-screen.md](../requirements/sales-screen.md)
> 본 문서는 **구현 독립적인 비즈니스 로직**을 정의한다. 화면/디자인은 다루지 않는다.

---

## A-1. 목적

일반 사용자가 현금을 투입하고 음료를 구매하는 전 과정의 비즈니스 규칙을 정의한다.
**한 세션 안에서 잔액을 유지하며 여러 번 구매**할 수 있으며, 특별 BM 규칙(프로모 할인·동적 가격·배출 실패)이 적용된다(A-7).

---

## A-2. 세션 & 상태 모델

### A-2.1 세션(Session) 정의
- 현금 투입으로 잔액이 생긴 시점에 **세션이 시작**되고, **잔액이 0이 되면 세션이 종료**된다.
- 잔액이 0이 되는 경로: (a) 구매로 잔액을 정확히 소진, (b) 수동/자동 반환으로 잔여 잔액을 반환.
- 세션 종료 시 세션 상태(구매 카운트 등, A-7.1)는 초기화된다.

### A-2.2 잔액(Balance) 모델
- 투입한 현금은 **잔액**에 누적된다. 구매 시 **거스름돈을 그때그때 반환하지 않고**, 잔액에서 최종가(A-7.2)만 차감한다.
- 남은 잔액으로 계속 구매할 수 있고, 잔여 잔액 반환은 **세션 종료 시**에만 이뤄진다(A-3.4).

### A-2.3 상태
| 상태 | 설명 |
|------|------|
| `IDLE` | 잔액 0, 대기(세션 없음) |
| `ACTIVE` | 잔액 > 0, 구매 가능(구매 사이 대기) |
| `DISPENSING` | 구매 확정 후 배출 시도(1~2초, 성공/실패 판정) |
| `RETURNING` | 잔여 잔액 반환 진행 |

전이:
```
IDLE ──투입──► ACTIVE ──상품선택(구매가능)──► DISPENSING ──(성공/실패)──► ACTIVE(잔액>0)
                │  ▲                                                        │
                │  └── 추가 투입(타이머 리셋)                               └── 잔액 0 ──► IDLE(세션 종료)
                └── 10초 미구매 / 반환버튼 ──► RETURNING(잔여 잔액) ──► IDLE(세션 종료)
```

## A-3. 기능 규칙

### A-3.1 상품 디스플레이
- 상품은 **최대 9개**(헌법 §3.2)이며, 화면에서는 **3×3 그리드**로 나열한다(UI: screen-layouts §2). 각 칸에 **상품 이미지(직사각 큐브)·재고·최종가(A-7.2)**를 노출한다.
- 구매 가능 여부:
  - `품절`: 재고 == 0 → **금액을 회색으로 표시하고 선택 불가**
  - `금액 부족`: 재고 있음 & 잔액 < 최종가
  - `구매 가능`: 재고 있음 & 잔액 >= 최종가
- 잔액·재고·**관리자의 실시간 수정**이 반영되면(동적 가격·프로모 포함) 각 상품의 최종가·구매 가능 여부를 **재평가**한다(INV-7 실시간 동기화).

### A-3.2 현금 투입
- 허용 화폐: **100 / 500 / 1,000원** (INV-3). 그 외는 거부하고 상태 불변.
- 투입 시 `잔액 += 액면가`, 상태 → `ACTIVE`.
- 투입 시 **자동 반환 타이머(10초)를 리셋**한다(BR-A7).

### A-3.3 상품 선택 & 배출
1. 선택 상품이 `구매 가능`인지 검증(아니면 거부: 품절/금액부족).
2. **최종가 산정**(A-7.2): `최종가 = max(0, 기본가 + 동적인상 − 프로모할인)`.
3. **결제(원자적, INV-4 원장 보존)**: `잔액 -= 최종가`, **세션 구매 카운트 += 1**.
4. 상태 → `DISPENSING`, **1~2초 지연** 후 **배출 판정**(A-7.3):
   - **성공(90%)**: 상품 배출, `재고 -1`, `매출 += 최종가`.
   - **실패(10%)**: 상품 미배출, `재고 -1`(상품 소실), **결제액 미반환(잔액 복구 안 함)**, **매출 불변** → 기계가 그 금액을 "배출 실패 미반환금"으로 보유.
5. 배출 판정 후: 잔액 > 0이면 `ACTIVE`(추가 구매 가능), 잔액 == 0이면 `IDLE`(세션 종료).

> 거스름돈은 구매마다 반환하지 않는다. 잔여 잔액은 세션 종료 시(A-3.4) 일괄 반환한다.

### A-3.4 잔여 잔액 반환 (최소 개수)
- 반환 금액 = **세션 종료 시 남은 잔액**(정확 소진 시 0, 수동/자동 반환 시 잔여 전액).
- **최소 개수**: 1,000 → 500 → 100 순 탐욕적 차감(INV-5). 반환분만큼 자판기 잔돈 차감(INV-6).
- **거스름돈 부족**(BR-A4): 최소 개수로도 정확히 만들 수 없으면 반환 예외 처리(A-6 가정).

### A-3.5 자동/수동 반환
- **자동 반환**: `ACTIVE`에서 **10초 미구매** → 잔여 잔액 반환 → `IDLE`(세션 종료).
- **수동 반환**: 반환 요청 시 즉시 잔여 잔액 반환 → `IDLE`(세션 종료).

## A-5. 예시 (Worked Examples)

- **다회 구매 세션 + 프로모**: 5,000 투입 → 사이다(1,200) 구매(잔액 3,800) → 생수(500) 구매(잔액 3,300) → **3번째 구매 −500**: 콜라(1,500→1,000) 구매(잔액 2,300) → 반환 버튼 → 2,300 최소개수 반환, 세션 종료.
- **동적 가격(개별)**: 콜라 재고가 3이 됨 → **콜라만** +500 → 콜라 2,000(붉은색), 생수 등 나머지는 불변(노란색).
- **동적+프로모 상쇄**: 같은 상품에 인상(+500) & 3번째 구매(−500) → 최종가 = 기본가(net 0).
- **배출 실패**: 잔액 2,000에서 콜라(1,500) 구매 → 결제로 잔액 500 → 배출 실패(10%) → 상품 없음, 잔액 500 복구 안 됨, **콜라 재고 -1**, 매출 불변, 미반환금 +1,500.
- **최소 개수**: 반환액 700 → 500×1 + 100×2 (총 3개).

## A-6. 미해결/가정 (Assumptions)

> 요구사항에 명시되지 않아 실제 자판기 관례/합리적 기본값으로 정한 결정(헌법 R7). **⚠️ 표시는 확정 필요.**

- **A6-1 동적 가격 범위** ✅확정: **재고 ≤3인 해당 상품만** +500(전역 아님, 상품별 독립). 인상 최종가는 붉은색, 평상시는 노란색.
- **A6-2 최종가 산정 순서**: `기본가 + 동적인상(500) − 프로모할인(500)`, 하한 0. 둘 다 해당 시 상쇄(net 0).
- **A6-3 프로모 카운트 기준**: 세션 내 **결제 발생 횟수**(배출 실패도 결제이므로 포함). 카운트 3·6·9…번째 구매 상품에 −500.
- **A6-4 배출 실패 처리** ✅확정: 실패 시 **재고 -1(상품 소실)·매출 미포함**, 결제액은 기계 보유(미반환).
- **A6-5 거스름돈 일괄 반환**: 구매마다 반환하지 않고 세션 종료 시 잔여 잔액 반환(신규 세션 모델 A-2). 기존 "구매마다 잔돈 반환 후 IDLE 초기화" 가정은 **폐기**.
- **A6-6 배출 성공 난수**: TDD를 위해 RNG는 **주입/시드 가능**해야 한다(테스트에서 성공·실패를 강제, §10).
- **A6-7 거스름돈 부족 정책** ✅확정(2026-07-02, 구현 결정): 세션 종료/반환 시 최소 개수로도 정확히 만들 수 없으면 **반환을 거부하고 잔액·세션을 그대로 유지**한다(부분 반환 금지). 사용자에게 "거스름돈 부족" 경고를 표시하고, 관리자가 잔돈을 보충하면 재시도 가능. 돈 손실이 없어 **INV-4 원장이 보존**된다. (구현: `domain/session.ts` refund → `INSUFFICIENT_CHANGE`, TC-A15)

## A-7. 특별 비즈니스 규칙 (Special BM)

> 원 요구사항의 특별 BM 상세 로직. 불변식/요약은 [spec.md](../spec.md) INV-4·§3.5 참조.

### A-7.1 세션 프로모 할인 (2+1 성격)
- 한 세션 안에서 **구매 2번마다 3번째 구매 상품을 500원 할인**한다.
- 즉 세션 구매 카운트 `n`이 **3의 배수(3, 6, 9, …)**일 때 그 구매 상품에 **−500(하한 0)** 적용.
- 카운트는 세션 종료(잔액 0) 시 초기화(BR-A12). 카운트 기준은 A6-3.

### A-7.2 재고 기반 동적 가격
- **재고가 3개 이하인 해당 상품만** 가격이 **500원 인상**된다(**전역 아님**, 상품별 독립).
- **최종가 = max(0, 기본가 + 동적인상 − 프로모할인)**.
- 인상은 조건 기반으로 재평가된다(재고가 3 초과로 회복되면 인상 해제).
- **색상 표기**: 인상된 최종가는 **붉은색**, 평상시 최종가는 **노란색**으로 표시한다(design-system §5.1).

### A-7.3 노후 자판기 배출 실패
- 배출은 **90% 확률로 성공**, **10% 확률로 실패**한다.
- **실패 시**: 상품 미배출 + **결제(사용) 금액 미반환** + **재고 -1(상품 소실)** + **매출 미포함**(판매 미완료).
- 즉 실패해도 재고는 차감되고 사용 금액도 잃는다. **기계 원장은 보존**된다(INV-4: 미반환금은 기계가 보유). 난수 정책은 A6-6.
