# [UI] Design System

> 상위 규범: [spec.md](../spec.md)
> 참조: [ui/screen-layouts.md](screen-layouts.md)
> 자판기 시뮬레이터의 시각/컴포넌트 규격. 모던 웹브라우저 기준.

---

## 1. 디자인 원칙

1. **직관성**: 실제 자판기처럼 "무엇을 눌러야 하는지"가 즉시 보인다.
2. **상태 가시성**: 투입 금액, 구매 가능 여부, 배출/반환 진행 상태가 항상 명확히 보인다.
3. **오조작 방지**: 구매 불가 상품/버튼은 시각적으로 비활성 처리한다.
4. **피드백 즉시성**: 투입·선택·배출·반환 각 동작에 즉각적 시각/텍스트 피드백을 준다.

## 4. 간격 & 형태

- 간격 스케일: `4 · 8 · 12 · 16 · 24 · 32 (px)`.
- 모서리: 카드 `12px`, 버튼 `8px`, 화폐 버튼 `999px(pill)`.
- 그림자: 카드 `0 2px 8px rgba(0,0,0,.35)`.

## 5. 컴포넌트 규격

### 5.1 상품 카드 (Product Card) — 3×3 그리드
- 판매 화면 **좌상단 3×3 그리드**(최대 9칸, 헌법 §3.2)에 배치한다(screen-layouts §2).
- 칸 구성: **상품 이미지(직사각 큐브 — 상품별 색 §5.10)·재고 수량·최종가**.
- 상태 표현:
  - `구매 가능` → `--color-success` 강조
  - `금액 부족` → `--color-warning`
  - `품절`(재고 0) → **금액을 회색(`--color-text-muted`)으로 표시하고 클릭 불가**(카드 비활성)
- **최종가 표시/색상**(특별 BM): 평상시 최종가는 **노란색(`--color-price`)**, **동적 인상(재고 ≤3인 그 상품) 시 붉은색(`--color-price-raised`)**. 프로모 할인 시 `원가 취소선 + 할인가` 강조. (최종가 = 기본가 + 동적인상 − 프로모할인, sales-mode A-7.2)
- 상태별 표현은 [business/sales-mode.md](../business/sales-mode.md) A-3.1을 따른다.

### 5.2 화폐 버튼 (Money Button)
- 100 / 500 / 1,000원 세 개(pill 형태).
- 누르면 해당 액면가 투입, 짧은 눌림 애니메이션 + 투입 금액 증가 반영.
- > 판매 화면에서는 **드래그 화폐 투입(§5.7)**이 기본 인터랙션이며, 이 버튼은 드래그 불가 환경을 위한 **접근성/키보드 대체 입력**으로 유지한다.

### 5.3 금액 디스플레이 (LED Display)
- 현재 투입 금액을 고정폭·발광 스타일로 표시(`--color-display`).
- 배출/반환 시 상태 메시지("배출 중...", "잔액 반환 중...", "배출 실패 — 금액 미반환")를 함께 표시.

### 5.4 액션 버튼
- **`반환` 버튼**: `--color-danger` 아웃라인 + **흰색 30% 배경(모든 상태)**. **투입 공간(§5.8) 바로 아래**에 상시 배치(screen-layouts §2.2 ⓡ). 누르면 잔여 잔액을 반환하고 세션을 종료한다(sales-mode A-3.5).
- `구매/선택` 확정: 상품 그리드(§5.1)의 칸 클릭으로 수행.

### 5.5 관리자 테이블
- 재고/잔돈/매출을 행 단위로 표시.
- 수량·개수·가격은 인라인 편집 가능한 숫자 입력 + `저장` 버튼.
- 저장 성공/실패 토스트 피드백.

### 5.6 토스트/알림
- 성공(success), 경고(warning), 오류(danger) 3종.
- 자동 반환 발생, 거스름돈 부족, 저장 완료 등에 사용.

### 5.7 드래그 화폐 (Draggable Money) — 판매 화면
- **자산**: `resources/100.png`, `resources/500.png`, `resources/1000.jpeg` (1000은 **jpeg** 확장자 주의 — §8).
- 화폐 트레이(screen-layouts §2.2 ④)에 칩 형태로 정렬하며 `draggable`. 각 칩은 **무한 소스**(드래그해도 트레이에서 사라지지 않음).
- drag payload로 **액면가(100/500/1000)** 메타데이터를 전달한다.
- 상태: hover 시 살짝 확대, dragging 시 반투명 고스트.
- 접근성: 클릭/Enter로도 투입(드래그 대체, §5.2와 동일 동작).
- **3D 렌더(현행, spec §8.3.2)**: 화폐 트레이는 3D 씬에 편입한다. **coin(100·500)** 은 **원통 윗면·아랫면에 `100.png`/`500.png`**, **bill(1,000)** 은 **얇은 직사각 큐브 윗면·아랫면에 `1000.jpeg`** 를 커버. 3D 화폐를 투입구로 드래그하거나 클릭하면 투입되며, 무한 소스(투입 후 원위치 복귀)로 동작한다.

### 5.8 투입 공간 (Money Drop Zone) — 판매 화면
- 위치/강조: screen-layouts §2.2 ①. 점선+발광 테두리 + "여기에 돈을 넣으세요" 라벨.
- **기본(not-hover) 배경**: 흰색 30% (`rgba(255,255,255,.3)`) — 반투명 device 위에서 가독성 확보.
- 드래그 오버: 테두리/배경 하이라이트(`--color-primary`)로 드롭 가능 표시.
- 유효 드롭: 액면가만큼 투입(→ §5.9 잔액 증가) + 흡입 모션. 허용 외 화폐/영역 밖 드롭: 원위치 복귀(no-op, INV-3).
- **3D 투입 허용 범위(현행, spec §8.3.2)**: 3D에서는 투입구 한 점이 아니라 **자판기 전체(본체·유리·패널·투입구·상품)** 를 유효 드롭 영역으로 둔다. 3D 화폐를 자판기 위 어디든 놓거나(레이가 자판기에 닿으면) **짧게 클릭**하면 투입된다. 자판기 밖(옆/바닥)에 놓으면 원위치 복귀. (좁은 판정으로 투입이 안 되던 이슈 보정)

### 5.9 잔액 표시 (Balance Display) — 판매 화면
- 위치: screen-layouts §2.2 ② (투입 공간 바로 위).
- 스타일: `--color-display` LED, `--font-display`, `₩` 접두. 값 변경 시 **카운트업** 애니메이션.
- 데이터 출처는 §5.3 LED 디스플레이와 동일한 **투입 금액(백엔드 상태)**이며, 판매 화면에서의 배치형이다.

### 5.10 배출 큐브 (Dispensed Product Cube) — 판매 화면
- 상품 배출 시 표시하는 **placeholder 직사각 큐브**(실제 상품 이미지 대신 사용).
- **색**: 상품마다 다른 색으로 구분(예: 콜라=`#E74C3C`, 사이다=`#27AE60`, 생수=`#2F80ED`, 커피=`#8B5E3C`) — 구분용 임의 색.
- **크기**: 배출 공간에 들어갈 정도(예: 64×88px).
- **모션**: **배출 공간 top-center에서 튀어나와** 회전하며 낙하 → 뷰포트 bottom 밀착 정지(타이밍/동작은 screen-layouts §6.3).
- 착지 시 하단 그림자로 "바닥에 붙음"을 강조.

## 6. 상태 표현 (State Styling)

| 상태 | 시각 처리 |
|------|-----------|
| 비활성(disabled) | opacity 0.4, 커서 not-allowed |
| 배출 중(dispensing) | 진행 스피너 + "배출 중..." (1~2초) |
| **배출 실패(dispense fail)** | danger 강조 + "배출 실패 — 사용 금액이 반환되지 않았습니다" 메시지 (특별 BM A-7.3) |
| 반환 중(returning) | 진행 스피너 + "반환 중..." |
| **선택 거부(reject)** | 구매 불가 상품(금액 부족·품절)을 선택하면 해당 상품 그래픽이 **붉게 깜빡이며(blink) 좌우로 흔들린다(shake)** + "금액 부족"/"품절" 토스트 (BR-A2·BR-A3, REQ-A10·A13). 3D에서는 큐브의 danger emissive 점멸 + 감쇠 좌우 진동으로 구현. |
| 타임아웃 임박 | 잔여 시간(10초) 카운트다운 표시 |

## 7. 프레임 / 반응형 / 접근성

- **고정 프레임(비하이브리드)**: 앱은 반응형으로 폭을 확장하지 않는다. **최대 width 500px**(초과 폭은 좌우 여백 두고 중앙 정렬), **height는 항상 뷰포트 최대치(100vh)** 유지. 모바일형 세로 프레임 (헌법 §8.3).
- 프레임 내부는 폭 500px 기준으로 배치하며, `device.png`는 프레임 폭에 맞춰 세로 비율을 유지한다.
- 버튼 최소 터치 영역 44×44px.
- 색만으로 상태를 구분하지 않고 **텍스트 라벨 병기**(색각 접근성).
- 상태 변화는 `aria-live`로 스크린리더에 전달.

## 8. 리소스 (Resources)

구현에 사용하는 이미지 자산. 경로는 **프로젝트 루트 기준**이며, 문서(`docs/ui/*`)에서의 상대경로는 `../../resources/…`.

| 자산 | 경로 | 용도 | 형식 |
|------|------|------|------|
| 자판기 본체 | `resources/device.png` | 판매 화면 중앙 배경(세로형) — screen-layouts §2 | PNG |
| 100원 | `resources/100.png` | 드래그 화폐(§5.7) | PNG |
| 500원 | `resources/500.png` | 드래그 화폐(§5.7) | PNG |
| 1,000원 | `resources/1000.jpeg` | 드래그 화폐(§5.7) | **JPEG** |

> ⚠️ `1000`은 확장자가 `.png`가 아니라 **`.jpeg`**다. 파일명을 하드코딩할 때 액면가별 확장자를 개별 지정할 것(예: 매핑 테이블 사용).
> 배출 상품에는 별도 이미지가 없으며, **색이 다른 직사각 큐브**로 대체한다(§5.10).
