## 2. 판매 화면 레이아웃

판매 화면은 **실제 자판기 사진 `resources/device.png`(세로형)을 브라우저 중앙에 크게 배치**하고,
그 위에 조작 영역을 오버레이하는 **스큐어모픽(skeuomorphic)** 구조다.
(이미지 경로는 프로젝트 루트 기준 `resources/…`, 문서에서의 상대경로는 `../../resources/…`.)

> **프레임 제약**: 전체 화면은 **고정 프레임(최대 width 500px, height 100vh, 비하이브리드)** 안에 배치된다(헌법 §8.3, [design-system §7](design-system.md)). device 이미지는 이 프레임 폭에 맞춰 세로 비율을 유지하며 중앙 정렬한다.
> **device 반투명화**: 오버레이 조작 영역의 가독성을 위해 device 이미지 위에 **흰색 20% foreground 오버레이**를 덧씌운다(오버레이 조작 요소는 그 위, `z-index` 상위).

### 2.1 전체 배치 개념도

```
┌──────────── 뷰포트 (고정 프레임 ≤500px · 중앙) ────────────┐
│        ┌──────────── resources/device.png ──────────┐      │
│        │ ⑤ 상품 그리드 3×3(좌상단)   ┌── 잔액 ──┐  │      │  ② 잔액 표시
│        │ ┌──┬──┬──┐  각 칸:         │  ₩ 0     │  │      │     (우상단·강조)
│        │ │큐│큐│큐│  이미지(큐브)   └──────────┘  │      │
│        │ ├──┼──┼──┤  + 재고         ┌──────────┐  │      │  ① 투입 공간
│        │ │큐│큐│큐│  + 최종가       │💰 여기에 │  │      │     (우측·드롭 타깃)
│        │ ├──┼──┼──┤ (품절=회색·불가)│  돈 넣기 │  │      │
│        │ │큐│큐│큐│                 └──────────┘  │      │
│        │ └──┴──┴──┘                 [   반환   ]  │      │  ← 반환 버튼(① 바로 밑)
│        │ ┌───────────┐      (PUSH)                 │      │  ③ 배출 공간
│        │ │③ 배출 공간│                             │      │     (좌하단·강조)
│        │ └───────────┘                             │      │
│        └───────────────────────────────────────────┘      │
│     ④ 화폐 트레이:  [🪙100]  [🪙500]  [💵1000]            │
│                                                            │
│  ⬇ 배출 시: 상품 큐브가 회전하며 낙하 → 뷰포트 bottom 밀착 (§6.3) │
└────────────────────────────────────────────────────────────┘
```

### 2.2 오버레이 영역 규격 (device 이미지 박스를 0~100% 좌표계로)

| # | 영역 | 위치(대략) | 강조 방식 | 근거 |
|---|------|-----------|-----------|------|
| ⑤ | **상품 그리드** (Product Grid) | 좌상단 · 3×3 (`top 6% / left 3% / w 60% / h 66%`) | 칸마다 상품 이미지(큐브)·재고·최종가. **품절 시 금액 회색·선택 불가**. 최대 9칸(헌법 §3.2) | 판매 ① |
| ① | **투입 공간** (Money Drop Zone) | 우측 중단 · `top 22% / right 3% / w 34% / h 22%` | 점선+발광 테두리, "여기에 돈을 넣으세요" 라벨, 드래그 오버 시 하이라이트 | 판매 ② |
| ② | **잔액 표시** (Balance Display) | ① 바로 위 · `top 8% / right 3% / w 34% / h 9%` | LED 스타일 대형 숫자(₩ 접두), 고대비 | 판매 ③ |
| ⓡ | **반환 버튼** (Refund) | ① **바로 아래** · `top 46% / right 3% / w 34% / h 7%` | "잔액 반환" 라벨 버튼(`--color-danger` 계열), 누르면 잔여 잔액 반환·세션 종료(A-3.5) | 판매(반환) |
| ③ | **배출 공간** (Dispense Zone) | 좌하단 · `bottom 3% / left 3% / w 44% / h 16%` | 어두운 개구부 + 강조 테두리, 배출 시 내부 발광 | 판매 ④ |
| ④ | **화폐 트레이** (Money Tray) | device 하단 바깥, 뷰포트 중앙 | `resources/100.png · 500.png · 1000.jpeg`를 드래그 가능한 칩으로 정렬(무한 소스) | 판매 ⑥ |

> 좌표는 **스펙 오버레이 값**이며 사진 속 실제 하드웨어 위치와 다를 수 있다(시뮬레이터). 값은 device 이미지 컨테이너 기준 상대 배치이며, 컴포넌트 상세는 [design-system](design-system.md) §5.1·5.4·5.7~5.10 참조.

### 2.3 상품 선택
- 상품 선택은 **⑤ 상품 그리드(3×3)**의 칸을 클릭해 수행한다. 각 칸은 상품 이미지(큐브)·재고·최종가를 표시하며, 구매 가능 여부·판정은 [business/sales-mode.md](../business/sales-mode.md) A-3.1을 따른다.
- **품절(재고 0)** 칸은 금액이 **회색**으로 표시되고 **클릭 불가**하다.
- 상품은 최대 9개이므로 그리드는 최대 3×3으로 채워진다(빈 슬롯은 비활성 표시, 헌법 §3.2 · admin BR-B10).

## 3. 관리자 화면 레이아웃

```
┌───────────────────────────────────────────────┐
│  관리자 모드                  [판매 모드로 ◂]  │
├───────────────────────────────────────────────┤
│  [ 재고 관리 ] [ 잔돈 관리 ] [ 매출 ] [로그*]  │  ← 탭
├───────────────────────────────────────────────┤
│  (선택 탭 콘텐츠)                               │
│  재고 관리 예 (상품 최대 9개, 기본가 100~99999):│
│   상품    기본가    수량    조작                │
│   콜라   [1500]   [ 8 ]   [저장]                │
│   사이다 [1200]   [ 5 ]   [저장]                │
│   ...                            [+ 상품 추가]  │
└───────────────────────────────────────────────┘
```
`*로그` 탭은 예정(planned) — 비활성 또는 "준비 중" 표시.
`[+ 상품 추가]`는 상품이 9개면 비활성(BR-B10). 기본가 입력은 100~99,999원으로 제한(BR-B6).

## 4. 화면 전환 흐름

```
        ┌──────────────┐   [관리자 진입]    ┌──────────────┐
        │  판매 화면    │ ─────────────────► │  관리자 화면  │
        │ (Sales)      │                    │ (Admin)      │
        │              │ ◄───────────────── │              │
        └──────────────┘   [판매 복귀]      └──────────────┘
```

- **모드 공존(INV-7 개정)**: 판매/관리자는 **다른 탭에서 동시에** 열 수 있다. 위 토글은 단일 탭 내 화면 전환일 뿐, 배타 제약이 아니다.
- **전환 시 세션 유지**: 관리자 진입이 진행 중인 판매 세션을 강제 종료하지 않는다(BR-B2).
- **실시간 반영**: 관리자 변경은 SSE로 **모든 판매 탭에 즉시 반영**된다(BR-B7·BR-B11) — 최종가·품절·색상 재평가.

## 6. 애니메이션 / 인터랙션 / 타이밍

### 6.1 공통
- 화폐 칩 눌림(드래그 대체 입력): 100~150ms.
- 자동 반환 카운트다운: `ACTIVE` 진입 시 10초부터 감소, 추가 투입 시 리셋(BR-A7).
- **반환 버튼(ⓡ)**: 투입 공간 바로 아래에 상시 표시. 클릭 시 잔여 잔액을 반환하고 세션을 종료한다(A-3.5).

### 6.2 화폐 드래그 투입 (요구 ⑥)
- ④ 화폐 트레이의 `100 / 500 / 1000` 이미지를 **드래그**하여 ① 투입 공간에 드롭한다.
- **드래그 오버**: 투입 공간 테두리/배경 하이라이트(`--color-primary`)로 드롭 가능함을 표시.
- **유효 드롭**: 해당 액면가만큼 **② 잔액 표시의 숫자가 증가**한다(카운트업 애니메이션 권장). 화폐 칩은 투입 공간으로 빨려들어가는 짧은 모션 후 사라진다.
- **금액 반영**: 투입 금액은 백엔드 상태(SSoT, R9)를 통해 갱신하며, **허용 화폐 100/500/1000만** 처리한다(INV-3). 투입 공간 밖/트레이 밖 드롭은 원위치 복귀(no-op).
- **접근성 대체**: 드래그 불가 환경에서는 화폐 칩 클릭/Enter로도 동일 투입(design-system §5.2·5.7).

### 6.3 상품 배출 낙하 애니메이션 (요구 ⑤)
- **트리거**: 구매 확정 → `DISPENSING` 상태(배출 지연 1~2초, 헌법 §3.3).
- **표현체**: 상품을 **색이 다른 직사각 큐브(placeholder)**로 렌더한다(실제 상품 이미지는 사용하지 않음). 큐브 규격은 design-system §5.10.
- **모션**:
  - 큐브가 **③ 배출 공간의 top-center 부근에서 튀어나온(pop)** 뒤 **회전(rotate)하며 아래로 낙하**한다.
  - **뷰포트 bottom(하단 가장자리)에 딱 붙는 위치**에서 정지한다(`position: fixed; bottom: 0`). 착지 x위치는 ③ 배출 공간 부근.
  - 낙하는 중력 느낌의 `ease-in` + 착지 시 소량 바운스(1회) 후 정지, 회전은 낙하 동안 지속하다 정지 각도로 정착.
- **완료(성공)**: 큐브가 정지하면 배출 완료로 간주한다. 잔액이 남아 있으면 `ACTIVE`(추가 구매), 0이면 세션 종료.
- **실패(10%, 특별 BM A-7.3)**: 낙하 큐브 없이 배출 공간이 잠깐 흔들리는 정도로만 처리하고, **"배출 실패 — 사용 금액이 반환되지 않았습니다"** danger 메시지를 표시한다(잔액은 이미 차감된 채 유지, design-system §6).
