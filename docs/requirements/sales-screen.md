# [Requirements] Screen A — 판매 화면 (Sales Screen)

> 상위 규범: [spec.md](../spec.md)
> 비즈니스 근거: [business/sales-mode.md](../business/sales-mode.md)
> UI 근거: [ui/screen-layouts.md](../ui/screen-layouts.md), [ui/design-system.md](../ui/design-system.md)
>
> 본 문서는 판매 화면의 **화면 요구사항**과 **테스트 케이스(TC)**를 포함한다.

---

## 1. 화면 목적

사용자가 현금을 투입하고, 한 세션 안에서 잔액으로 음료를 (여러 번) 구매하며, 잔여 잔액 반환을 확인하는 화면.

---

## 2. 화면 요구사항 (Functional Requirements)

| ID | 요구사항 | 근거 |
|----|----------|------|
| REQ-A1 | 상품 카드에 상품명·**최종가**·구매가능여부(가능/금액부족/품절)를 표시한다. | sales-mode A-3.1 |
| REQ-A2 | 100/500/1,000원 화폐를 투입(드래그 또는 버튼)해 잔액을 올릴 수 있다. | INV-3, A-3.2 |
| REQ-A3 | 현재 **잔액**을 디스플레이에 실시간 표시한다. | A-2, UI 5.9 |
| REQ-A4 | 잔액·재고 변화 시 각 상품의 최종가·구매가능여부를 즉시 갱신한다. | A-3.1 |
| REQ-A5 | 구매 가능한 상품 선택 시 **최종가 결제 → 배출 시도(1~2초, 성공/실패)**를 수행하고 잔액을 차감한다(거스름돈은 세션 종료 시 반환). | A-3.3 |
| REQ-A6 | **세션 종료 시** 잔여 잔액을 최소 개수 화폐로 반환한다. | INV-5, A-3.4 |
| REQ-A7 | 반환 버튼으로 잔여 잔액을 즉시 반환하고 세션을 종료한다. | A-3.5 |
| REQ-A8 | 미구매 상태 10초 경과 시 잔여 잔액을 자동 반환한다. | A-3.5 |
| REQ-A9 | 추가 투입 시 10초 자동반환 타이머를 리셋한다. | BR-A7 |
| REQ-A10 | 품절/금액부족 상품은 비활성 처리하고 선택을 거부한다. | BR-A2, BR-A3 |
| REQ-A11 | 세션 종료 시 거스름돈 부족으로 정확 반환 불가하면 예외 처리한다. | BR-A4 |
| REQ-A12 | 페이지 새로고침/백엔드 재시작 후에도 재고·매출·(세션 잔액·구매 카운트) 상태가 유지된다. | INV-1 |
| REQ-A13 | 배출/반환/**배출 실패** 진행·결과를 시각·텍스트로 피드백한다. | UI 6 |
| REQ-A14 | 세션 내 **3·6·9…번째 구매 상품에 500원 할인**을 적용해 표시·청구한다. | A-7.1 |
| REQ-A15 | **재고 3개 이하인 해당 상품만** 가격을 **500원 인상**해 표시·청구하며, 인상가는 **붉은색**·평상시는 **노란색**으로 표시한다. | A-7.2, design-system §5.1 |
| REQ-A16 | 배출은 **90% 성공/10% 실패**이며, **실패 시 결제액 미반환 + 재고 -1**(매출 불변)이다. | A-7.3, INV-4 |
| REQ-A17 | 잔액이 0이 되면 세션을 종료하고, 세션 내 **다회 구매**를 지원한다. | A-2 |
| REQ-A18 | 뷰포트는 **고정 프레임(최대 width 500px, height 100vh, 비하이브리드)**로 렌더한다. | 헌법 §8.3, design-system §7 |
| REQ-A19 | 상품을 좌상단 **3×3 그리드**에 **이미지(직사각 큐브)·재고·최종가**로 표시하고, **반환 버튼을 투입 공간 바로 아래**에 배치한다. | sales-mode A-3.1, UI §2 |
| REQ-A20 | **관리자 수정(재고·가격)이 판매 화면에 실시간 반영**되며, 여러 탭이 동시에 열려 있어도 상태가 동기화된다. | INV-2, INV-7 |

## 3. 테스트 케이스 (Test Cases)

> 형식: **전제(Given) → 행동(When) → 기대결과(Then)**. 각 TC는 위 REQ와 매핑. 배출 성공/실패가 관여하는 TC는 RNG를 **강제(주입/시드)**한다(sales-mode A6-6).

### Feature: 현금 투입
| TC | 매핑 | Given | When | Then |
|----|------|-------|------|------|
| TC-A01 | REQ-A2,A3 | IDLE, 잔액 0 | 1,000원 투입 | 디스플레이 ₩1,000, 상태 ACTIVE |
| TC-A02 | REQ-A2 | 잔액 1,000 | 500 + 100 추가 투입 | 디스플레이 ₩1,600 |
| TC-A03 | BR-A1 | 임의 상태 | 허용 외 화폐(예: 5,000) 시도 | 거부, 잔액 불변 |

### Feature: 상품 구매 가능 여부
| TC | 매핑 | Given | When | Then |
|----|------|-------|------|------|
| TC-A04 | REQ-A1,A4 | 잔액 0 | 화면 조회 | 전 상품 "금액 부족" |
| TC-A05 | REQ-A4 | 잔액 500, 생수 재고>3 | 화면 갱신 | 생수 "구매 가능", 나머지 "금액 부족" |
| TC-A06 | REQ-A10 | 콜라 재고 0 | 화면 조회 | 콜라 "품절" — **금액 회색·클릭 불가** |

### Feature: 구매 & 배출(성공 경로)
| TC | 매핑 | Given | When | Then |
|----|------|-------|------|------|
| TC-A07 | REQ-A5,A17 | 잔액 1,500, 콜라 재고>3, RNG=성공 | 콜라 선택 | 1~2초 후 배출, 재고-1, 매출+1,500, 잔액 0 → 세션 종료(IDLE) |
| TC-A08 | REQ-A5,A17 | 잔액 2,000, 콜라(1,500), RNG=성공 | 콜라 선택 | 배출 성공, 잔액 500 남음, 상태 ACTIVE(추가 구매 가능) |
| TC-A09 | REQ-A6 | 세션 종료 시 반환액 700 | 잔액 반환 | 500×1 + 100×2 (총 3개, 최소 개수) |
| TC-A10 | BR-A2 | 잔액 500 | 콜라(1,500) 선택 | 선택 거부, "금액 부족" |
| TC-A11 | BR-A3 | 생수 재고 0 | 생수 선택 | 선택 거부, "품절" |

### Feature: 반환 (수동/자동)
| TC | 매핑 | Given | When | Then |
|----|------|-------|------|------|
| TC-A12 | REQ-A7 | 잔액 1,600, 미구매 | 반환 버튼 클릭 | 즉시 1,600 최소개수 반환, 세션 종료(IDLE) |
| TC-A13 | REQ-A8 | 잔액 500, 미구매 | 10초 대기 | 500 자동 반환, 세션 종료 |
| TC-A14 | REQ-A9 | 잔액 투입 후 9초 경과 | 100원 추가 투입 | 타이머 10초로 리셋, 자동반환 지연 |
| TC-A15 | REQ-A11 | 잔돈 500·100 보유 0, 1,000 투입(→잔돈 편입) 후 생수 500 구매 → 잔액 500 반환 필요하나 500/100 없음 | 반환 시도 | 반환 예외 처리(정책 A6-7, A6-8 반영) |

### Feature: 특별 BM — 프로모 할인 (A-7.1)
| TC | 매핑 | Given | When | Then |
|----|------|-------|------|------|
| TC-A16 | REQ-A14 | 세션서 2회 구매 완료(카운트=2), 잔액 충분, RNG=성공 | 3번째로 콜라(1,500) 구매 | 최종가 1,000(−500) 청구, 매출+1,000 |
| TC-A17 | REQ-A14 | TC-A16 이후 세션 종료 → 새 세션 | 다시 첫 구매 | 카운트 1부터, 할인 미적용 |

### Feature: 특별 BM — 동적 가격 (A-7.2)
| TC | 매핑 | Given | When | Then |
|----|------|-------|------|------|
| TC-A18 | REQ-A15 | 콜라 재고를 3으로 만듦 | 화면 조회 | **콜라만** 2,000(붉은색), 생수 등 나머지는 기본가(노란색) — 개별 적용 |
| TC-A19 | REQ-A14,A15 | 같은 상품에 재고 조건(+500) & 3번째 구매(−500) | 해당 상품 구매 | 최종가 = 기본가(net 0) |

### Feature: 특별 BM — 배출 실패 (A-7.3)
| TC | 매핑 | Given | When | Then |
|----|------|-------|------|------|
| TC-A20 | REQ-A16 | 잔액 2,000, 콜라(1,500) 재고 5, RNG=**실패** | 콜라 선택 | 상품 미배출, 잔액 500(복구 안 됨), **재고 4로 -1**, 매출 불변, 미반환금 +1,500 |
| TC-A21 | REQ-A16 | 동일, RNG=**성공** | 콜라 선택 | 배출 성공, 재고 4로 -1, 매출+1,500, 잔액 500 |

### Feature: 세션 & 영속성

|----|------|-------|------|------|
| TC-A22 | REQ-A17 | 잔액이 구매로 정확히 0 | 마지막 구매 후 | 상태 IDLE, 세션 구매 카운트 초기화 |
| TC-A23 | REQ-A12 | 세션 잔액 1,000·구매 카운트 2 진행 중 | 페이지 새로고침/백엔드 재시작 | 잔액·카운트·재고·매출 유지 |

|----|------|-------|------|------|

## 4. 완료 기준 (Acceptance)

- REQ-A1 ~ A20 전부 충족, TC-A01 ~ A29 전부 Pass.
- INV-1·2·3·5·6 위반 0건. **INV-4(개정) 원장 보존** 항상 성립(TC-A24).
- 특별 BM의 ⚠️ 가정(sales-mode A6-1·A6-4 등)은 확정 후 해당 TC 기대값을 재확인.
