# [Requirements] Screen B — 관리자 화면 (Admin Screen)

> 상위 규범: [spec.md](../spec.md)
> 비즈니스 근거: [business/admin-mode.md](../business/admin-mode.md)
> UI 근거: [ui/screen-layouts.md](../ui/screen-layouts.md), [ui/design-system.md](../ui/design-system.md)
>
> 본 문서는 관리자 화면의 **화면 요구사항**과 **테스트 케이스(TC)**를 포함한다.

---

## 1. 화면 목적

운영자가 재고·잔돈·매출을 관리하는 화면(감사 로그는 예정). 판매 모드와 **동시에 열릴 수 있고**, 변경은 판매 화면에 실시간 반영된다.

---

## 2. 화면 요구사항 (Functional Requirements)

| ID | 요구사항 | 근거 |
|----|----------|------|
| REQ-B1 | 재고 목록(상품명·기본가·수량)을 조회한다. | admin-mode B-3.1 |
| REQ-B2 | 상품 수량을 보충/차감한다(결과 음수 불가). | BR-B4 |
| REQ-B3 | 상품 기본가를 **100~10,000원** 범위로 변경한다(범위 밖 거부, 경계 허용). | BR-B6 |
| REQ-B12 | 상품은 **최대 9개**까지 추가할 수 있다(초과 추가 거부). | BR-B10 |
| REQ-B4 | 잔돈 보유량을 화폐단위(1,000/500/100)별 개수·총액으로 조회한다. | B-3.2 |
| REQ-B5 | 잔돈을 단위별로 보충/회수한다(결과 음수 불가). | BR-B5 |
| REQ-B6 | 누적 매출액(및 상품별 판매 현황)을 조회한다. | B-3.3 |
| REQ-B7 | 관리자 변경은 즉시 판매 디스플레이에 반영된다. | BR-B7 |
| REQ-B8 | 관리자 변경은 영속 저장되어 재시작/새로고침 후 유지된다. | INV-1, BR-B9 |
| REQ-B9 | 관리자 모드와 판매 모드는 **동시에 열릴 수 있다**(다른 탭, 배타 아님). | INV-7, BR-B1 |
| REQ-B10 | 관리자 진입은 진행 중 판매 세션을 **강제 종료하지 않는다**(진입 시 강제 반환 없음). | BR-B2 |
| REQ-B11 | 감사 로그 탭은 "예정"으로 노출하되 데이터 적재 구조를 설계한다. | B-3.4 |
| REQ-B13 | 관리자 변경(재고·가격·잔돈)은 **모든 판매 탭에 실시간(SSE) 반영**된다. | INV-7, BR-B7, BR-B11 |

## 3. 테스트 케이스 (Test Cases)

### Feature: 재고 관리
| TC | 매핑 | Given | When | Then |
|----|------|-------|------|------|
| TC-B01 | REQ-B1 | 초기 시드 데이터 | 재고 탭 조회 | 콜라1500·사이다1200·생수500·커피1800 및 각 수량 표시 |
| TC-B02 | REQ-B2 | 콜라 수량 0 | 콜라 수량 5로 보충·저장 | 저장 성공, 수량 5 |
| TC-B03 | REQ-B2,B7 | TC-B02 이후 | 판매 화면 이동 | 콜라 "품절"→"구매가능/금액부족"으로 전환 |
| TC-B04 | BR-B4 | 콜라 수량 2 | 수량 -5(→-3) 시도 | 거부, 수량 불변 |
| TC-B05 | REQ-B3 | 콜라 1500 | 가격 1600으로 변경·저장 | 저장 성공, 판매 화면 가격 1600 반영 |
| TC-B06 | BR-B6 | 임의 상품 | 가격 50 또는 10,500 시도 | 거부(범위 100~10,000), 값 불변 |
| TC-B17 | BR-B6 | 임의 상품 | 가격 100 / 10,000(경계) 저장 | 허용 |
| TC-B18 | REQ-B12, BR-B10 | 상품 9개 등록 상태 | 10번째 상품 추가 시도 | 거부(최대 9개) |
| TC-B19 | REQ-B12 | 상품 8개 | 9번째 상품 추가 | 허용, 총 9개(3×3 그리드 가득) |

## 4. 완료 기준 (Acceptance)

- REQ-B1 ~ B10, B12, B13 충족, TC-B01 ~ B20 Pass.
- REQ-B11/TC-B16은 데이터 적재 구조 확인까지 (조회 UI는 이후 단계).
- INV-6 위반 0건. INV-7(모드 공존·실시간 동기화) 충족. 상품 개수 ≤ 9, 기본가 ∈ [100, 10,000] 항상 성립.
